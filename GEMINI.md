# GEMINI

你正在开发一个生产级 Vue 前端项目。

项目技术栈：

- Vue 3
- Vite
- TailwindCSS
- TypeScript

项目目标：

基于提供的设计稿，实现一个现代化 Minecraft 社区官网。

设计稿已经提供，文件名为`design.png`，请严格参考设计稿进行开发。设计稿提炼出的要求可以参考`DESIGN.md`。设计稿内所有图片元素均在`public/images`文件夹内。

---

网站风格：

- Minecraft 社区
- 游戏感

- 科技感

- 大面积留白

视觉关键词：

- 现代 Landing Page
- 类似游戏社区官网

# UI 还原要求

必须：

- 高度还原设计稿
- 保持布局结构一致
- 保持视觉层级一致
- 保持间距一致
- 保持字体大小比例一致
- 保持按钮风格一致

禁止：

- 擅自重新设计页面
- 擅自修改配色
- 擅自增加复杂装饰
- 擅自更换整体风格

---

# 页面结构要求

当前页面包含：

- 顶部导航栏
- Hero Banner
- 玩法介绍区域
- 管理团队区域
- 加入服务器 CTA 区域
- Footer

开发时请拆分为独立组件。

---

# 组件拆分规范

建议目录结构：

src/
  components/
    navbar/
    hero/
    features/
    staff/
    cta/
    footer/

  pages/
  layouts/
  assets/
  styles/

要求：

- 每个区域拆分为独立组件
- 卡片组件可复用
- 保持组件职责单一
- 避免超大组件

---

# Vue 开发规范

必须：

- 使用 Composition API
- 使用 `<script setup>`
- 使用 TypeScript
- Props 显式定义类型

禁止：

- Options API
- 大量全局状态
- 单文件超大组件

---

# TailwindCSS 规范

优先：

- Tailwind 工具类
- 响应式工具类
- Flex/Grid 布局

避免：

- 大量自定义 CSS
- 复杂 SCSS
- 过多 arbitrary values

---

# 响应式要求

必须支持：

- Desktop
- Tablet
- Mobile

要求：

- 禁止横向滚动
- 保持统一边距
- 卡片自动换行
- Hero 区域移动端自适应

移动端允许：

- 部分区域纵向排列
- Navbar 折叠菜单

---

# 动画规范

允许：

- Hover 动效
- 渐变过渡
- 轻量淡入动画
- 按钮微交互

禁止：

- 复杂炫技动画
- 影响性能的特效
- 高频动画

动画目标：

自然、现代、克制。

---

# 图片处理规范

必须：

- 使用合理 object-fit
- 保持图片比例
- 避免图片拉伸
- 优化 Banner 显示效果

Hero 背景图：

- 优先使用 cover
- 保持主体可见
- 必要时添加遮罩提升文字可读性

---

# 性能要求

必须：

- 保持组件轻量
- 避免不必要依赖
- 避免重复渲染
- 保持较高 Lighthouse 分数

禁止：

- 无意义大型库
- 复杂运行时逻辑

---

# 文件修改规范

禁止：

- 修改无关文件
- 重构整个项目
- 擅自修改 package.json
- 删除已有组件

除非明确要求。

---

# 代码质量要求

必须：

- 保持代码可读性
- 保持组件可维护
- 删除死代码
- 控制复杂度
- 保持目录清晰

---

# 完成任务前必须检查

开发完成前必须确认：

- 页面接近设计稿
- Build 成功
- 无 TypeScript 报错
- 无 Console 报错
- 响应式正常
- 动画自然
- 无横向滚动
- 图片显示正常
